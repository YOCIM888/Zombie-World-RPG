# 📋 末日生存游戏 — 数据维护指南

## 核心原则

- **游戏数据按模块组织在 `js/data/` 文件夹中**，各子模块负责不同类型的数据
- **`js/config.js` 作为统一的重新导出入口**，所有数据通过 `config.js` 对外暴露，其他文件只需从 `config.js` 导入
- 修改 `js/data/` 下的数据文件后刷新浏览器即可生效
- 其他文件（`combat.js`、`maps.js`、`state.js` 等）不需要修改
- 新增物品只需在对应子模块的数组中添加条目
- `config.js` 和 `utils.js` 内部有些常量是自动派生的（如 `NURSE_MEDICINE_POOL`、`V_TRADE_AMMO_TYPES`、`BUILDING_MATERIAL_NAMES`、`BOSS_NAMES`），修改源数据后自动更新

## 二、常见操作指南

### 2.1 新增食物

在 `FOODS` 数组末尾添加条目即可。

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 唯一标识，全局唯一 |
| `name` | string | ✅ | 显示名称 |
| `type` | string | ✅ | 固定为 `"food"` |
| `hunger` | number | ✅ | 恢复饱腹值 |
| `hydration` | number | ❌ | 恢复水分值，省略则不恢复水分 |
| `rarity` | string | ❌ | 稀有度：`common`/`uncommon`/`rare`/`epic`/`legendary` |

**示例：添加"红烧肉"**

```js
// 在 FOODS 数组末尾添加
{ id: "红烧肉", name: "红烧肉", type: "food", hunger: 50, hydration: 10, rarity: "uncommon" },
```

---

### 2.2 新增近战武器

在 `MELEE_WEAPONS` 数组末尾添加条目。

**字段说明：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 唯一标识 |
| `name` | string | ✅ | 显示名称 |
| `type` | string | ✅ | 固定为 `"melee"` |
| `damage` | number | ✅ | 基础伤害值 |
| `durability` | number | ✅ | 最大耐久度（拳头为 `Infinity`） |
| `rarity` | string | ❌ | 稀有度 |
| `comboRate` | number | ✅ | 连击概率（0~1） |

**示例：添加"链锯剑"**

```js
{ id: "链锯剑", name: "链锯剑", type: "melee", damage: 110, durability: 90, rarity: "epic", comboRate: 0.16 },
```

**稀有度与连击率对应关系：**

| 稀有度 | 连击率 | 伤害范围参考 |
|--------|--------|-------------|
| common | 0.05 | 30~48 |
| uncommon | 0.08 | 40~70 |
| rare | 0.12 | 66~90 |
| epic | 0.15 | 99~105 |
| legendary | 0.18 | 118+ |

---

### 2.3 新增远程武器

添加远程武器需要**同时关注弹药关联**。

**步骤1：在 `RANGED_WEAPONS` 添加武器**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 唯一标识 |
| `name` | string | ✅ | 显示名称 |
| `type` | string | ✅ | 固定为 `"ranged"` |
| `damage` | number | ✅ | 基础伤害值 |
| `integrity` | number | ✅ | 完整度（通常为100） |
| `ammoType` | string | ✅ | 使用的弹药类型 |
| `critRate` | number | ✅ | 暴击概率（0~1） |
| `rarity` | string | ✅ | 稀有度 |

**步骤2：如果需要新弹药类型，在 `AMMO` 添加弹药**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✅ | 弹药唯一标识 |
| `name` | string | ✅ | 显示名称 |
| `type` | string | ✅ | 固定为 `"ammo"` |
| `compatibleWith` | string[] | ✅ | 兼容武器ID列表 |

**示例：添加"SCAR-H"**

```js
// 1. 在 RANGED_WEAPONS 末尾添加
{ id: "SCAR-H", name: "SCAR-H", type: "ranged", damage: 85, integrity: 100, ammoType: "7.62×51mm", critRate: 0.28, rarity: "rare" },

// 2. 在 AMMO 中找到 "7.62×51mm"，添加兼容武器
{ id: "7.62×51mm", name: "7.62×51mm", type: "ammo", compatibleWith: ["M700", "FAL", "EBR14", "SR25", "SCAR-H"] },
```

## 三、`js/data/` 文件夹结构与扩展指南

### 3.1 文件夹组织原则

`js/data/` 文件夹按数据类型划分子目录，每个子目录包含：

- **数据文件**：定义具体的常量和数据
- **`index.js`**：统一导出该目录下的所有常量

### 3.2 导出链路

```
数据文件 (如 foods.js)
  → 子目录 index.js (如 items/index.js)
    → js/data/index.js
      → js/config.js (重新导出 + 工具函数)
        → 其他模块 (如 game.js, combat.js 等)
```

### 3.3 如何添加新的数据文件

**步骤1：创建数据文件**

```js
// js/data/items/crops-ext.js
export const SPECIAL_CROPS = [
  { id: "变异种子", name: "变异种子", matureTurns: 10, reward: { food: "万能针剂" } },
];
```

**步骤2：在子目录的 `index.js` 中添加导出**

```js
export { SPECIAL_CROPS } from './crops-ext.js';
```

### 3.4 如何添加新的子目录

如果需要创建全新的数据分类（如 `js/data/quests/`）：

1. 创建子目录和文件
2. 在 `js/data/index.js` 中添加导出
3. 在子目录的 `index.js` 中导出常量

## 四、稀有度体系说明

游戏使用6级稀有度体系：

| 稀有度 | 英文 | 中文 | 掉落权重 | 颜色参考 |
|--------|------|------|---------|---------|
| common | common | 普通 | 20 | 白/灰 |
| uncommon | uncommon | 优秀 | 12 | 绿 |
| rare | rare | 稀有 | 6 | 蓝 |
| epic | epic | 史诗 | 3 | 紫 |
| legendary | legendary | 传奇 | 1 | 橙/金 |
| myth | myth | 神话 | 0.5 | 红 |

**注意：** `myth` 稀有度物品不会出现在随机掉落中，仅用于剧情奖励。

## 六、注意事项

1. **id 必须全局唯一** - 每个物品的 `id` 在其所属数组内必须唯一
2. **修改 id 会影响存档兼容性** - 建议只修改 `name` 字段，保持 `id` 不变
3. **新增物品不需要修改其他文件** - 掉落系统、作弊系统会自动识别
4. **删除物品需要检查所有引用** - 检查 `FIXED_LOOT_DROPS`、`DEFAULT_ITEM_IDS`、`AMMO.compatibleWith` 等
5. **rarity 字段必须是有效值** - `common`、`uncommon`、`rare`、`epic`、`legendary`、`myth`
6. **弹药与武器必须同步** - `ammoType` 和 `compatibleWith` 必须保持一致
7. **新增地图需添加丧尸池映射** - 在 `MAP_ZOMBIE_POOL_MAP` 中添加映射
8. **作弊命令格式** - `/get_{type}_{id}_{count}`，例如 `/get_food_压缩饼干_5`
